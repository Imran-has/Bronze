# Skill: Summary

## Purpose

Summarize documents, notes, or content into concise overviews.

## When to Use

- Task requests a summary
- Task mentions "summarize", "condense", "overview", "key points"
- Need to distill long content into shorter form

## Instructions

1. **Read source material**
   - Read the entire document(s)
   - Note the main topic and purpose

2. **Identify key points**
   - Main arguments or ideas
   - Important facts or data
   - Conclusions or recommendations

3. **Determine summary length**
   - Brief: 1-2 paragraphs
   - Standard: 3-5 paragraphs
   - Detailed: Full page with sections

4. **Write summary**
   - Start with main point/conclusion
   - Include supporting key points
   - Maintain original meaning
   - Use clear, concise language

5. **Verify accuracy**
   - Check against source
   - Ensure no misrepresentation

## Output Format

```markdown
# Summary: [Document Title]

**Source**: [original file]
**Date**: [summary date]

## Key Points

- Point 1
- Point 2
- Point 3

## Overview

[Summary paragraphs]
```

## Example

**Task**: "Summarize the meeting notes from Monday"

**Output**: A summary highlighting decisions made, action items, and key discussion points
